## Discovery — Brain Dump (2026-06-22)

### D1: Chat-First Entry
User lands in chat mode immediately after login. Chat is the default view and primary interaction surface.

### D2: Inline System/Repo Selection
Options for selecting repositories or systems are shown within the chatbox area — not on a separate page or modal. Selection is contextual to the conversation.

### D3: Dual-Theme Mode Switching
- Business Mode = lighter UI theme
- IT Mode = darker UI theme
- Mode selector positioned above the chatbox
- Switching modes starts a **new chat session** — mid-session mode switching is not supported
- Each chat session is locked to one mode; theme follows the session

### D4: Navigation to Management
A link positioned at the edge of the chat page allows navigation from chatting mode to management screens. This is a peripheral navigation element, not the primary action.

### D5: Management — System Tree View
In the management screen, System Owners see all systems they own listed as a tree view with repositories nested underneath each system.

### D6: ~~Hidden Knowledge Graph~~ **RETRACTED — see D16**

### D7: Dashboard on Repo Page
Scan progress monitoring lives within the repository detail/page context. When a scan is running for a given repo, the user can navigate to a dashboard view via an icon or entrance element on that repo's page.

### D8: HITL Notification Center
HITL questions are aggregated in a notification center, exposed through a notification icon (bell or similar) positioned in the corner of the page. This notification icon is visible on both the chat surface and the management surface — globally accessible within the app shell.

### D9: Language
UI language is Chinese (Simplified). Single language for V1, no i18n surface design.

### D10: ~~No Document Browsing UI~~ **RETRACTED — see D17**

### D11: Visual Personality — Warm Professional
Vulgata feels like a trusted internal tool with warmth. It sits on the Fluent UI skeleton but adds a warm anchor color — professional enough for a bank, approachable enough for non-technical users. Not cold like a terminal, not playful like a consumer app.

### D13: Chosen Theme — Library Oak
- **Primary:** `#445E7A` (light) / `#6A83A2` (dark) — warm blue-gray
- **Accent:** `#B98B6B` (light) / `#D4A88C` (dark) — oak tan
- **Background light:** `#F6F4F0` (warm paper)
- **Background dark:** `#1D2026` (deep warm charcoal)
- **Register:** Scholarly, contemplative, wise — deep warm blue-grays with oak-toned accents. Reads like a reading room, not a terminal.

### D14: Typography — Chinese-Friendly Override
- **Body / UI:** Fluent UI default (Segoe UI on Windows, system fallback). No override.
- **Display:** Noto Serif SC (思源宋体) for hero moments — empty-chat greeting, section headers in management, mode-select banner when starting a new session. The serif matches the scholarly Library Oak register and reads naturally in Chinese.
- Used sparingly: first-session greeting, new-chat prompt, empty states. Never for body text, buttons, labels, or data.

### D15: Rounded Corners — Fluent UI Defaults
Keep Fluent UI defaults: 4px (small), 8px (medium), 12px (large). No override. Fluent's native radius already feels right for the warm-professional register.

### D16: Live Knowledge Graph — Visible (retracts D6)
The live knowledge graph IS directly visible to users via a dedicated Graph page. Uses Z.Blazor.Diagrams. Nodes appear as documents are generated; cross-repo links appear as edges. Two layout modes: Hierarchical and Force-directed. Edge styling per architecture spec (intra-repo solid, cross-repo dashed, uncertainty as dashed-to-placeholder). Animations (200–400ms fade). Viewport preserved across updates.

### D17: Document Browsing UI — Reinstated (retracts D10)
Users have access to document list and document detail views. `DocumentsPage` and `DocumentDetailPage` from architecture spec are reinstated. Documents organized in virtual tree mirroring source directory hierarchy. Two types visible: Code Logic Documents and Business Logic Documents.

### D18: Management Surface — Holy Grail Layout
The management surface uses a holy grail layout:
- **Top:** Navbar with brand, management menu items (系统管理, 图谱, 文档, 扫描历史, 设置), notification bell, user avatar
- **Left sidebar:** Context-sensitive navigation (system tree in 系统管理, document tree in 文档, etc.)
- **Main content:** Selected system/repo details, graph view, document view, etc.
- System/Repo creation and editing is accessible via buttons/actions in the main content area and sidebar.
- Management entrance from Chat: navbar link "管理后台".

### D19: System & Repo Management Entrances
- **Create System:** "+ 新建系统" button in the system management sidebar or empty state
- **Create Repository:** "+ 新建仓库" button in system detail's repo list or empty state
- **Edit System/Repo:** Edit button in the detail header for each system/repo
- **Delete:** Confirm dialog triggered from detail page actions menu

### D20: Less-Important Pages
User management (Admin/Users), LLM provider config (Admin/Providers), user profile, login/register, and scan history pages follow common Fluent UI patterns — standard CRUD forms, FluentDataGrid for lists, FluentDialog for confirmations. No unique UX design needed; inherit Fluent defaults with Library Oak brand colors.
