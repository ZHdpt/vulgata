# EXPERIENCE.md — vulgata

> 行为规范。组件行为、状态模式、交互原语、关键流程。视觉规格见 `DESIGN.md`。
> Component library: Microsoft Fluent UI Blazor。Blazor Interactive Server 渲染模式。

## Foundation

**形态：** Web 应用，桌面浏览器。Blazor Interactive Server，通过 SignalR 维持实时连接。单容器 Docker 部署。

**UI 系统：** Microsoft Fluent UI Blazor。组件库承担大部分基础行为；品牌层覆盖仅限 `DESIGN.md` 定义的 tokens。圆角沿用 Fluent 默认值（4px / 8px / 12px），无覆盖。

**视觉参考：** `DESIGN.md` 为视觉识别权威来源。本文档中 `{DESIGN.md.colors.primary}` 语法表示对 `DESIGN.md` 中定义的设计 token 的交叉引用。

**约束：** 无离线状态设计。无 i18n — 仅简体中文。无 WCAG 专项合规要求。V1 单语言、单部署。

## Information Architecture

### App Shell（应用壳）

登录后所有界面共享 **顶栏导航**（Navbar）。顶栏包含：
- 左侧：品牌标识 + 主导航菜单项（对话、管理后台）
- 右侧：通知铃铛图标（带未读计数徽章）+ 用户头像（下拉：个人资料、退出登录）

**Chat 界面** 和 **管理后台** 是两个顶层路由，通过顶栏导航切换。

### 管理后台 — Holy Grail 布局

管理后台使用 holy grail 布局（顶栏 + 左侧栏 + 主内容区）：

```
┌──────────────────────────────────────────┐
│  顶栏: 品牌 | 系统管理 | 图谱 | 文档 | ...│
├──────────┬───────────────────────────────┤
│ 左侧栏    │  主内容区                       │
│ (导航树)  │  选中系统/仓库的详情             │
│          │  或图谱视图                      │
│ 系统列表  │  或文档视图                      │
│  ├ 系统A  │                                │
│  │ ├仓库1 │                                │
│  │ └仓库2 │                                │
│  └ 系统B  │                                │
│          │                                │
└──────────┴───────────────────────────────┘
```

**顶栏菜单项（管理后台）：**
| 菜单项 | 左侧栏内容 | 主内容区 |
|---|---|---|
| **系统管理** | 系统树视图（系统 → 仓库） | 选中系统/仓库的详情 + 扫描状态 |
| **图谱** | — | 全屏知识图谱（Z.Blazor.Diagrams） |
| **文档** | 文档树（虚拟目录结构） | 选中文档的完整内容 |
| **扫描历史** | 扫描记录列表 | 选中扫描的详细结果 |
| **设置** | 设置导航 | LLM 配置 / 用户管理（Admin） |

### 各界面详情

| 界面 | 入口 | 用途 |
|---|---|---|
| 登录/注册 | 首次访问 | 邮箱+密码注册；登录后直接进入 Chat 界面 |
| Chat（对话） | 顶栏"对话" | 主要交互界面。模式选择、系统/仓库选择、对话区 |
| 管理后台 → 系统管理 | 顶栏"管理后台" → "系统管理" | 系统/仓库 CRUD，扫描进度监控 |
| 管理后台 → 图谱 | 顶栏"管理后台" → "图谱" | 实时知识图谱可视化 |
| 管理后台 → 文档 | 顶栏"管理后台" → "文档" | 文档树浏览 + 文档详情 |
| 管理后台 → 扫描历史 | 顶栏"管理后台" → "扫描历史" | 历史扫描记录 |
| 管理后台 → 设置 | 顶栏"管理后台" → "设置" | LLM Provider 配置、用户管理（Admin） |
| 通知中心 | 全局铃铛图标 | HITL 问题汇总，滑出面板 |

**Chat 界面结构（从上到下）：**
1. 顶栏（品牌 + "对话"/"管理后台" 导航 + 铃铛 + 头像）
2. 模式选择器（业务模式 / 技术模式 双标签切换）
3. 内联系统/仓库选择器
4. 对话区（消息气泡流）
5. 输入框（底部固定）

**管理后台结构（从上到下，左到右）：**
1. 顶栏（品牌 + 菜单项：系统管理 / 图谱 / 文档 / 扫描历史 / 设置 + 铃铛 + 头像）
2. 左侧栏（内容随菜单项变化：系统树 / 文档树 / 扫描列表 / 设置菜单）
3. 主内容区（选中项的详细信息）

**系统管理页（默认管理后台着陆）：**
- 左侧栏：系统树视图 + "新建系统" 按钮
- 主内容区：
  - 选中系统时：系统名称、描述、仓库数量、扫描中仓库数、仓库列表（每行含扫描状态 + 仪表盘入口）、"新建仓库" 按钮、编辑/删除系统操作
  - 选中仓库时：仓库详情（名称、描述、git URL、上次扫描、文档数量）、扫描状态区（进度条 + 阶段指示器 + 仪表盘入口）、编辑/删除仓库操作

**通知中心：** 点击铃铛图标滑出右侧面板。HITL 问题卡片列表。全局可达 — 所有界面铃铛图标位置一致。

**文档浏览：** 文档按虚拟目录树排列（镜像仓库源码结构）。两种文档类型：代码逻辑文档、业务逻辑文档。点击文档查看完整 Markdown 内容。文档也可通过 Chat 引用访问。

**知识图谱：** 独立图谱页面，使用 Z.Blazor.Diagrams 渲染。两种布局模式：层级布局（镜像源码树）、力导向布局（揭示跨仓库模式）。节点 = 文档，边 = 引用关系。跨仓库边为虚线彩色，未解析边为虚线 + 菱形占位节点。动画 200–400ms。视口（缩放/平移）在更新时保持。

## Voice and Tone

### 业务模式

用户为非技术角色（PM、BA）。文案要求：平实、叙述性、无术语。

| ✅ 应写 | ❌ 不应写 |
|---|---|
| "风险评估涉及三个系统：手机银行接收用户请求，风控引擎执行规则校验，核心系统完成资金操作。" | "风控引擎通过 SOFA RPC 调用核心系统的 evaluateRisk 接口。" |
| "需要我帮你梳理哪个业务流程？" | "请输入查询参数。" |
| "找到了 3 份相关文档。" | "检索返回 3 条记录。" |
| "以下是贷款审批的完整步骤。" | "Business Logic Document #42 摘要如下。" |

### 技术模式

用户为开发者。文案要求：精确、代码感知、直接。

| ✅ 应写 | ❌ 不应写 |
|---|---|
| "riskEvaluate() 通过 Dubbo 调用 riskeng-service 的 evaluate() 方法 → `RiskController.java:156`" | "系统之间有一个调用关系。" |
| "跨仓库链路已解析：3 条确认，1 条待扫描。" | "链接情况还行。" |
| "该函数依赖 `UserService.validate()`（同仓库）和 `RiskClient.evaluate()`（跨仓库 → riskeng-service）" | "这个函数调用了其他一些东西。" |

### 通知

中性、可操作。不庆祝，不责备。

| ✅ | ❌ |
|---|---|
| "有 2 个待回答的问题。" | "🎉 新问题来啦！" |
| "风控引擎仓库有 1 个问题待确认。" | "错误：无法解析 — 请立即处理！" |

### 错误状态

不归咎。陈述事实 + 提供下一步。

| ✅ | ❌ |
|---|---|
| "连接失败，正在重试。" | "API 调用出错！" |
| "找不到相关文档。试试换个问法，或检查系统选择范围。" | "无结果。" |
| "扫描已暂停（LLM API 不可用）。API 恢复后自动继续。" | "扫描失败。" |

## Component Patterns

行为规范。视觉规格见 `DESIGN.md.Components`（或 Fluent UI 默认值）。

### ChatArea（对话区）

| 属性 | 规则 |
|---|---|
| 消息气泡 | 用户消息右对齐，系统消息左对齐。流式响应时逐字呈现。 |
| 引用 | 每条系统回答包含引用链接。点击引用展开内联预览（文档摘要 + 来源代码位置）。不跳转到独立文档页。 |
| 模式徽章 | 当前会话模式显示在对话区顶部：业务模式为浅色背景的 "业务模式" 标签，技术模式为深色背景的 "技术模式" 标签。 |
| 系统标签 | 当前选中的系统/仓库以小型 Fluent Tag 显示在输入框上方。可点击 × 移除。 |
| 空状态 | 新会话时居中显示 Noto Serif SC 欢迎语。"你好，想了解什么业务流程？"（业务模式）或 "你好，需要追踪哪段逻辑？"（技术模式）。 |
| 输入框 | `Enter` 发送，`Shift+Enter` 换行。placeholder 随模式变化。 |

### ModeSelector（模式选择器）

| 属性 | 规则 |
|---|---|
| 外观 | 双标签切换器，位于对话区上方、系统选择器上方。当前选中标签高亮为 `{DESIGN.md.colors.primary}`。 |
| 切换行为 | 切换模式时弹出确认："切换模式将开始新对话，当前对话将被保存。确定切换？" — 确认后清空对话区，显示新会话欢迎语，更新主题。 |
| 主题联动 | 业务模式 → 浅色主题；技术模式 → 深色主题。切换 200-400ms 过渡动画。 |

### InlineSelector（内联系统/仓库选择器）

| 属性 | 规则 |
|---|---|
| 外观 | 位于模式选择器下方、对话区上方。Fluent Dropdown 多选组件，以内联方式呈现。 |
| 选项 | 列出当前用户有权访问的所有系统，展开系统可看到其下仓库。支持全选系统、单独选仓库。 |
| 默认 | 新会话默认选中用户上次使用的选择范围。首次使用：全选所有有权限的系统。 |

### SystemTree（系统树视图）

| 属性 | 规则 |
|---|---|
| 外观 | Fluent TreeView。系统为父节点，仓库为子节点。展开/折叠箭头。选中节点高亮为 `{DESIGN.md.colors.accent}`。顶部有 "+ 新建系统" 按钮。 |
| 可见性 | 仅 System Owner 角色可见。普通用户无管理后台入口。 |
| 交互 | 点击系统节点 → 主内容区显示系统详情（系统信息、仓库列表、扫描中仓库数）。点击仓库节点 → 主内容区显示仓库详情。右键系统节点可编辑/删除。 |
| 空状态 | "尚未添加任何系统。创建你的第一个系统开始扫描。" + 主按钮"创建系统"。 |
| 扫描指示 | 仓库旁显示扫描状态点：灰（未扫描）、绿脉动（扫描中）、绿（已完成）、红（失败）。 |

### RepoDetail（仓库详情 — 主内容区）

| 属性 | 规则 |
|---|---|
| 内容 | 顶部：仓库名称 + 描述 + 编辑/删除操作按钮。信息卡片：git URL、上次扫描时间、扫描状态、文档数量。 |
| 扫描入口 | 扫描未运行时显示"开始扫描"主按钮和"查看文档"按钮（如果有已生成文档）。扫描运行中显示进度条 + 阶段指示器（内联，不弹出新页面）+ 脉动"仪表盘"链接。 |
| 编辑 | System Owner 可通过编辑按钮修改仓库元数据。 |

### SystemDetail（系统详情 — 主内容区）

| 属性 | 规则 |
|---|---|
| 内容 | 顶部：系统名称 + 描述 + 编辑/删除操作按钮。统计卡片：仓库总数、扫描中仓库数、已完成仓库数、文档总数。 |
| 仓库列表 | FluentDataGrid 表格：仓库名称、扫描状态、上次扫描时间、文档数。每行有"查看"操作。表头有 "+ 新建仓库" 按钮。 |

### GraphView（知识图谱）

| 属性 | 规则 |
|---|---|
| 入口 | 管理后台顶栏 → "图谱" 菜单项。 |
| 渲染 | Z.Blazor.Diagrams 全屏渲染。顶部悬浮工具栏：布局切换（层级 / 力导向）、缩放控制、系统/仓库筛选下拉框。 |
| 节点 | 文档节点。代码逻辑文档（蓝色）、业务逻辑文档（绿色）。失败文档红边框标记。 |
| 边 | 仓库内引用 = 实线浅灰。跨仓库引用 = 虚线彩色。未解析引用 = 虚线 + 红色菱形占位节点。争议边 = 黄色虚线 + 警告图标。 |
| 动画 | 新节点淡入，移除节点淡出（200–400ms）。视口（缩放/平移）在更新时保持。 |
| 空状态 | "尚无扫描数据。启动一次扫描后，知识图谱将在此展示。" |

### DocumentTree（文档浏览）

| 属性 | 规则 |
|---|---|
| 入口 | 管理后台顶栏 → "文档" 菜单项。 |
| 左侧栏 | 虚拟目录树，按仓库源码目录层级组织文档。两种文档类型标签：代码逻辑文档（蓝标签）、业务逻辑文档（绿标签）。 |
| 主内容区 | 选中文档的完整 Markdown 渲染内容。顶部：文档标题、类型、所属仓库/Source 路径。 |
| 搜索 | 左侧栏顶部搜索框，按文档标题/关键词过滤。 |
| 空状态 | "尚无已生成的文档。启动一次扫描后，文档将在此展示。" |

### ScanDashboard（扫描仪表盘）

| 属性 | 规则 |
|---|---|
| 入口 | 仓库详情中扫描运行时的脉动"仪表盘"链接（内联或独立页均可）。 |
| 内容 | 实时进度（文件已处理/总数）、活跃 Worker 数、错误计数、阶段指示器。SignalR 推送更新。 |
| 阶段指示器 | 水平步骤条：预扫描分析 → 代码逻辑文档生成 → 业务逻辑文档生成 → 后处理。 |

### NotificationCenter（通知中心）

| 属性 | 规则 |
|---|---|
| 触发 | 点击顶栏铃铛图标。滑出右侧面板，宽度 400px。 |
| 内容 | HITL 问题卡片列表。每条：仓库名称、问题摘要、时间戳、"回答"/"忽略"操作按钮。 |
| 状态 | 铃铛图标带红色数字徽章 = 未读问题数。无未读时徽章隐藏。 |
| 空状态 | "暂无待回答问题。扫描过程中产生的疑问会出现在这里。" |

### EmptyState（空状态）

| 场景 | 文案 | 排版 |
|---|---|---|
| 新会话（业务模式） | "你好，想了解什么业务流程？" | Noto Serif SC, `{DESIGN.md.typography.display}` |
| 新会话（技术模式） | "你好，需要追踪哪段逻辑？" | Noto Serif SC, `{DESIGN.md.typography.display}` |
| 无系统（系统管理） | "尚未添加任何系统。创建你的第一个系统开始扫描。" | 正文 + 主按钮"创建系统" |
| 空仓库列表（系统详情） | "该系统下暂无仓库。添加第一个仓库。" | 正文 + 主按钮"新建仓库" |
| 无通知 | "暂无待回答问题。扫描过程中产生的疑问会出现在这里。" | 正文 |
| 无图谱数据 | "尚无扫描数据。启动一次扫描后，知识图谱将在此展示。" | 正文 + 主按钮"去系统管理" |
| 无文档 | "尚无已生成的文档。启动一次扫描后，文档将在此展示。" | 正文 + 主按钮"去系统管理" |

## State Patterns

### LoadState 枚举

全局统一状态枚举：

| 状态 | 界面表现 |
|---|---|
| `Loading` | Fluent Shimmer（骨架屏），4-6 行，匹配预期布局。 |
| `Refreshing` | 内容保持可见，顶部细进度条（indeterminate），不阻塞交互。 |
| `Empty` | 引导文案 + 单一主操作按钮（见 EmptyState 表）。 |
| `NoResults` | "未找到匹配结果。试试调整关键词或系统选择范围。" |
| `Error` | 不归咎文案 + 重试按钮。业务模式不展示技术细节，技术模式可展示简化错误信息。 |

### 模式切换状态

| 状态 | 处理 |
|---|---|
| 切换确认 | 弹出 Fluent Dialog："切换模式将开始新对话。当前对话将被保存。确定切换？" |
| 切换中 | 主题过渡动画 200-400ms。对话区清空，显示新会话欢迎语。 |

### 扫描状态（RepoDetail）

| 状态 | 表现 |
|---|---|
| 未扫描 | 灰点 + "未扫描" 文字。显示"开始扫描"按钮。 |
| 扫描中 | 绿点（脉动）+ "扫描中…" + 进度百分比 + 仪表盘入口图标。 |
| 已完成 | 绿点 + "已完成" + 上次扫描时间戳。 |
| 失败 | 红点 + "失败" + 重试按钮 + 错误摘要。 |

### Chat 加载状态

| 状态 | 表现 |
|---|---|
| 等待回复 | 打字指示器（三点跳动），200ms 间隔。 |
| 检索中 | "正在查找相关文档…" 临时提示。 |
| 流式响应 | 逐字呈现，光标闪烁。 |
| 无结果 | "未找到相关文档。试试调整系统选择范围或换个问法。" |

## Interaction Primitives

### 键盘

Chat 界面键盘优先：`Enter` 发送消息，`Shift+Enter` 换行，`Esc` 关闭通知面板/弹窗。管理界面鼠标优先。

### 鼠标

- 点击引用链接 → 内联展开文档摘要预览（不跳转）
- 点击铃铛图标 → 滑出通知面板
- 悬停仓库行 → 显示操作按钮
- 所有操作即时反馈，无拖拽（V1）

### 模式切换

- 点击模式标签 → 弹出确认 Dialog → 确认后清空对话 + 切换主题 + 显示欢迎语
- 无撤销 — 确认即执行

### 扫描入口

- 扫描未运行 → 按钮"开始扫描"
- 扫描运行中 → 进度指示器 + 仪表盘图标（脉动）
- 扫描完成 → 图标消失，状态显示"已完成"

### Chat 引用

- 每条系统回答带引用链接
- 点击引用 → 内联展开：文档标题 + 摘要 + 来源代码位置行号
- 不跳转到独立页面

### 禁止项

- 无无限滚动（分页或"加载更多"）
- 无拖拽重排
- 无模态堆叠（弹窗不超过一层）
- 无自动刷新对话区

## Key Flows

### 林 — 产品经理想了解风险评估流程

1. 林打开 Vulgata，用邮箱和密码登录，直接进入对话界面。
2. 她在模式选择器中确认当前是「业务模式」（默认）。
3. 她在内联选择器中勾选「移动银行系统」的相关仓库。
4. 她打字："风险评估的完整流程是什么？" 按 Enter 发送。
5. 系统先显示"正在查找相关文档…"，随后流式输出回答：一步一步的业务叙述，纯中文、无代码、引用业务逻辑文档。
6. 她追问了一个细节，系统继续用业务语言回答。

**高潮节拍：** 林想确认完整链路。她结束当前对话，点击「新建对话」并把模式切换到「技术模式」。主题变为深色。她重新输入同样的问题，系统返回带代码文件路径和行号的完整技术追踪。她把两次回答放在一起比对，从业务和技术两面完全理解了流程。

### 陈 — 新入职业务分析师探索系统

1. 陈入职第一周，打开 Vulgata 登录进入对话界面（业务模式，浅色）。
2. 她在内联选择器中浏览可选系统，勾选了「核心银行系统」。
3. 她打字："贷款审批是怎么走的？"
4. 对话引导她理解相关系统和文档 — 每一步都是中文业务叙述，无代码。
5. 她点击回答中的文档引用，看到业务逻辑文档的摘要。

**高潮节拍：** 陈理解了整个审批流程后，尝试问了一个她自己的问题："如果客户信用评级是 C 级呢？" — 系统依然用业务语言回答，她意识到 Vulgata 可以成为她日常工作的助手。

### 王 — 开发追踪跨仓库调用链路

1. 王切换模式到「技术模式」— 主题立刻变为深色。
2. 他选中「移动银行系统」相关仓库，输入："风险评分计算涉及哪些跨仓库调用？"
3. 系统返回带代码引用的完整追踪 — `RiskEngine.evaluate() → LoanService.processApproval() → LedgerService.record()`，每个标注了仓库名、文件路径和通信类型（RPC/消息队列/HTTP）。
4. 他点击引用查看具体代码行，了解了完整的跨仓库依赖关系。

**高潮节拍：** 王追问："这段逻辑在哪个仓库里改了最近一次？" 系统返回了最近的代码变更和关联文档，他连文件都不用打开就确认了重构范围。

### 张 — 系统负责人配置扫描并处理 HITL 问题

1. 张登录后点击顶栏「管理后台」→「系统管理」。
2. 左侧栏显示他负责的系统树视图。他点击「核心银行系统」，主内容区显示系统详情：3 个仓库，1 个扫描中，2 个已完成。
3. 他点击主内容区的「+ 新建仓库」按钮，在弹出的表单中填写 git URL 和描述，保存。
4. 新仓库出现在左侧树和仓库列表中。他点击该仓库，主内容区切换到仓库详情。
5. 他点击「开始扫描」按钮。按钮变为进度条 + 阶段指示器，旁边出现脉动的「仪表盘」链接。
6. 他点击「仪表盘」，看到实时扫描进度 — 文件已处理/总数、活跃 Worker 数。
7. 铃铛图标出现红色未读计数。张点开通知中心，看到 HITL 问题："`LoanService.processApproval` 是外部调用还是内部逻辑？" 他回答后徽章计数减一。
8. 扫描完成。张切到「图谱」菜单，看到新节点出现在知识图谱中。再切到「文档」，浏览生成的代码逻辑文档和业务逻辑文档。

**高潮节拍：** 张切回「对话」，用业务模式问了一句："核心银行系统的贷款审批流程是什么？" — 系统基于刚才的扫描结果返回了完整、准确的业务叙述。
